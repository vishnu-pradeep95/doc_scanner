  <name>Task 1: Add R8 log stripping rules to ProGuard config</name>
  <files>app/proguard-rules.pro</files>
  <action>
**Append the following rule block to `app/proguard-rules.pro`** (after the existing rules, at end of file):

```proguard
# ===== SEC-03: Strip verbose/debug/info log calls from release builds =====
# R8 removes the method calls AND associated string concatenation (AGP 7.3+ / R8 3.3.70+).
# Log.w and Log.e are intentionally retained for crash diagnostics.
# Only effective when isMinifyEnabled = true (release builds).
-assumenosideeffects class android.util.Log {
    public static int v(...);
    public static int d(...);
    public static int i(...);
}
```

**Key facts (do not change):**
- R8 (AGP 8.13.2) handles StringBuilder/string concatenation removal automatically since AGP 7.3+
- Kotlin 1.9.21 generates StringBuilder bytecode that R8 optimizes correctly
- This only takes effect in minified release builds (`isMinifyEnabled = true`, already set in `build.gradle.kts` release buildType)
- Log.w (3 calls in codebase) and Log.e (15 calls) are intentionally preserved for crash diagnostics
- Use `(...)` wildcard for parameters — matches all overloads (v(String,String), v(String,String,Throwable))

**Anti-pattern to avoid:** Do NOT use wildcard `*` method matching (e.g., `* *(...)`) -- this can strip methods from `Object` superclass, causing runtime crashes. Always use explicit method names `v(...)`, `d(...)`, `i(...)`.

  </verify>

**1. Create `app/src/main/res/xml/network_security_config.xml`** (new file):

<!-- Defense-in-depth for offline app — prevents any component from making unencrypted connections -->
