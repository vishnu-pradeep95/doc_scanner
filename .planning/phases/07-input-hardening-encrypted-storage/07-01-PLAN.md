<objective>
Create centralized input validation and wire it into all fragment entry points to prevent path traversal attacks on navigation args and MIME type attacks on imported files.

Purpose: SEC-07 requires all external input (nav arg file paths and imported content URIs) to be validated before use. Path traversal via crafted `../` nav args could access files outside the app sandbox. Unvalidated MIME types could lead to processing unexpected content.

Output: InputValidator utility with unit tests, validation wired into PreviewFragment, PdfViewerFragment, PdfEditorFragment (path validation), and HomeFragment (MIME validation).
</objective>

  <action>
Create `InputValidator.kt` as an `object` in the `util/` package (follows ImageUtils, PdfUtils pattern).

**isPathWithinAppStorage(path: String, context: Context): Boolean**
- Return false for empty/blank paths
- Use `File(path).canonicalPath` to resolve `../` traversals and symlinks
- Compare against `context.filesDir.canonicalPath` — path must start with `filesDir + File.separator` OR equal filesDir exactly
- Catch IOException from canonicalization and return false (invalid path = rejected)

**isUriPathWithinAppStorage(uriString: String, context: Context): Boolean**
- Parse with `Uri.parse(uriString)`
- For `file://` scheme: extract `uri.path`, delegate to `isPathWithinAppStorage`
- For `content://` scheme: return true (ContentResolver mediates access)
- For null/unknown schemes: return false

**isAllowedMimeType(context: Context, uri: Uri): Boolean**
- Get MIME via `context.contentResolver.getType(uri)` — return false if null
- Reject `application/octet-stream` explicitly (per user decision: banking-app security stance)
- Accept `mimeType.startsWith("image/")` or `mimeType == "application/pdf"`
- Reject everything else

**Unit tests:** Use Robolectric (consistent with existing test patterns: `@RunWith(RobolectricTestRunner::class)`, `@Config(sdk = [34])`). For path tests, use `ApplicationProvider.getApplicationContext()` to get a real `filesDir`. For MIME tests, use `ShadowContentResolver` to register MIME types for test URIs.
