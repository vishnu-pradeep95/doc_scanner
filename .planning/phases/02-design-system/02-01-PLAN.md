**1c. themes_cartoon.xml — fix dark mode textColor in TextAppearance styles**

The following TextAppearance styles have `android:textColor` hardcoded to a non-night-aware color. Replace the direct color reference with a theme attribute so dark mode works correctly.

In `TextAppearance.Cartoon.BodyMedium`:
- Remove: `<item name="android:textColor">@color/cartoon_text_secondary</item>`
- Add: `<item name="android:textColor">?attr/colorOnSurfaceVariant</item>`

In `TextAppearance.Cartoon.BodySmall`:
- Remove: `<item name="android:textColor">@color/cartoon_text_secondary</item>`
- Add: `<item name="android:textColor">?attr/colorOnSurfaceVariant</item>`

In `TextAppearance.Cartoon.HeadlineLarge`, `HeadlineMedium`, `HeadlineSmall`, `TitleLarge`, `TitleMedium`, `TitleSmall`, `BodyLarge`:
- Remove: `<item name="android:textColor">@color/cartoon_text_primary</item>`
- Add: `<item name="android:textColor">?attr/colorOnSurface</item>`

Do NOT change LabelLarge, LabelMedium, LabelSmall — they already have no textColor override.

For each layout file, do the following two passes:

**Pass A — textSize to textAppearance:**
Find every `android:textSize="Xsp"` attribute in a TextView. Add `android:textAppearance` with the appropriate Cartoon style (using the mapping below), then REMOVE the `android:textSize` line — if both exist, textSize wins and defeats the purpose.

Size → style mapping:
  32sp → TextAppearance.Cartoon.HeadlineLarge
  26sp → TextAppearance.Cartoon.HeadlineMedium
  22sp → TextAppearance.Cartoon.HeadlineSmall
  20sp → TextAppearance.Cartoon.TitleLarge
  16sp Bold/SemiBold → TextAppearance.Cartoon.TitleMedium
  14sp SemiBold → TextAppearance.Cartoon.TitleSmall
  16sp Regular → TextAppearance.Cartoon.BodyLarge
  14sp Regular → TextAppearance.Cartoon.BodyMedium
  13sp → TextAppearance.Cartoon.BodySmall
  12sp → TextAppearance.Cartoon.BodySmall
  11sp → TextAppearance.Cartoon.LabelSmall

If a TextView already has `android:textAppearance`, do NOT add a duplicate — skip it.

**Pass B — hardcoded margins/paddings to dimens:**
Find `android:layout_margin`, `android:padding`, `android:paddingStart`, `android:paddingEnd`, `android:paddingTop`, `android:paddingBottom` with inline dp values. Replace with @dimen/ references:
  4dp  → @dimen/spacing_xxs
  8dp  → @dimen/spacing_xs
  12dp → @dimen/spacing_sm
  16dp → @dimen/spacing_md
  24dp → @dimen/spacing_lg
  32dp → @dimen/spacing_xl

Focus on fragment_settings.xml (has the most textSize violations) and item_*.xml files. For the fragment layouts, prioritize the key TextViews (titles, subtitles, body text) over minor decoration views. Skip RecyclerView divider heights and icon sizes — those are not spacing grid concerns.

Do NOT modify dialog_*.xml files — lower priority, handle if time permits.
