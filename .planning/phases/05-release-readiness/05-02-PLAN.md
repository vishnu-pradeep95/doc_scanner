       ```kotlin
       lint {
           abortOnError = true
           lintConfig = file("lint.xml")
           htmlReport = true
           htmlOutput = file("${project.buildDir}/reports/lint/lint-results.html")
       ```

       IMPORTANT: Use `lint {}` NOT `lintOptions {}` — `lintOptions` is deprecated in AGP 7.0+ (project uses AGP 8.13.2).

       ```xml
           <issue id="ContentDescription" severity="error" />

           <!-- Accessibility: touch targets smaller than 48dp -->
           <issue id="TouchTargetSizeCheck" severity="error" />
       </lint>
       ```

<task type="auto">
  <name>Task 2: Harden AndroidManifest.xml — camera uses-feature, backup exclusions, FileProvider scope</name>
  <files>app/src/main/AndroidManifest.xml, app/src/main/res/xml/data_extraction_rules.xml, app/src/main/res/xml/backup_rules.xml, app/src/main/res/xml/file_paths.xml</files>
  <action>
    Make three targeted changes to AndroidManifest.xml and two new XML resource files:

    **Change 1 — camera uses-feature (RELEASE-05):**
    In `app/src/main/AndroidManifest.xml`, find line:
    ```xml
    <uses-feature android:name="android.hardware.camera" android:required="true" />
    ```
    Change `android:required="true"` to `android:required="false"`.
    This makes the app installable on tablets and Chromebooks without cameras.

    **Change 2 — backup exclusion attributes (RELEASE-06):**
    On the `<application>` tag in `AndroidManifest.xml`, ADD two new attributes:
    ```xml
    android:dataExtractionRules="@xml/data_extraction_rules"
    android:fullBackupContent="@xml/backup_rules"
    ```
    Keep all existing attributes (`android:allowBackup="true"`, `android:label`, etc.) — only add these two.

    Create `app/src/main/res/xml/data_extraction_rules.xml` (API 31+ backup exclusions):
    ```xml
    <?xml version="1.0" encoding="utf-8"?>
    <data-extraction-rules>
        <cloud-backup>
            <!-- Exclude private scan directories — contain user document data -->
            <exclude domain="file" path="scans/" />
            <exclude domain="file" path="processed/" />
            <exclude domain="file" path="pdfs/" />
            <!-- cacheDir is automatically excluded by Android per AOSP backup documentation.
                 No explicit rule is needed for cache/ — Android handles it. -->
        </cloud-backup>
        <device-transfer>
            <exclude domain="file" path="scans/" />
            <exclude domain="file" path="processed/" />
            <exclude domain="file" path="pdfs/" />
            <!-- cacheDir is automatically excluded by Android — no explicit rule needed. -->
        </device-transfer>
    </data-extraction-rules>
    ```

    Create `app/src/main/res/xml/backup_rules.xml` (API 30 and below backup exclusions):
    ```xml
    <?xml version="1.0" encoding="utf-8"?>
    <full-backup-content>
        <exclude domain="file" path="scans/" />
        <exclude domain="file" path="processed/" />
        <exclude domain="file" path="pdfs/" />
        <!-- cacheDir (domain="external-cache") is automatically excluded by Android
             per AOSP backup documentation. No explicit rule is needed for cache/. -->
    </full-backup-content>
    ```

    **Change 3 — FileProvider cache-path scope (RELEASE-07):**
    In `app/src/main/res/xml/file_paths.xml`, change the cache-path entry:
    ```xml
    <!-- BEFORE (overly broad — exposes entire cacheDir subtree): -->
    <cache-path name="cache" path="/" />

    <!-- AFTER (scoped to cacheDir root only): -->
    <cache-path name="cache" path="." />
    ```
    Do NOT change the other paths (`files-path` entries for scans/, processed/, pdfs/).

    Note on why `path="."` is safe: All code that writes to cacheDir (PdfEditorFragment, NativePdfView, PdfUtils) writes directly to `context.cacheDir` root (e.g., `File(cacheDir, "temp_edit_*.pdf")`). None of them pass cache file paths to `FileProvider.getUriForFile()` — all FileProvider sharing uses the files-path entries. The `path="."` change is a tightening improvement; it covers the cacheDir root identically to what the app actually uses.
  </action>
