2. Create `app/src/main/java/com/pdfscanner/app/util/AppLockManager.kt` as an `object` singleton following the SecurePreferences pattern. Implementation per 09-RESEARCH.md Pattern 1:

   - All keys use `SecurePreferences.APP_PREFIX` prefix (e.g., `"${SecurePreferences.APP_PREFIX}lock_enabled"`)
   - Constants: `KEY_LOCK_ENABLED`, `KEY_LOCK_TIMEOUT`, `KEY_LAST_BACKGROUND_TIME`
   - Timeout constants: `TIMEOUT_IMMEDIATE = 0L`, `TIMEOUT_30_SECONDS = 30_000L`, `TIMEOUT_1_MINUTE = 60_000L`, `TIMEOUT_5_MINUTES = 300_000L`
   - `isLockEnabled(context: Context): Boolean` -- reads from SecurePreferences, default false
   - `setLockEnabled(context: Context, enabled: Boolean)` -- writes to SecurePreferences
   - `getTimeout(context: Context): Long` -- reads timeout, default TIMEOUT_IMMEDIATE
   - `setTimeout(context: Context, timeoutMs: Long)` -- writes timeout
   - `recordBackgroundTime(context: Context)` -- writes System.currentTimeMillis()
   - `shouldRequireAuth(context: Context): Boolean` -- returns false if not enabled; if enabled, checks elapsed time against timeout; returns true on first launch with lock (lastBackground == 0L)
   - `canAuthenticate(context: Context): Boolean` -- uses BiometricManager.from(context).canAuthenticate(getAllowedAuthenticators()); returns true if BIOMETRIC_SUCCESS or BIOMETRIC_ERROR_NONE_ENROLLED
   - `getAllowedAuthenticators(): Int` -- CRITICAL API-level split:
     * API 30+ (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R): `BIOMETRIC_STRONG or DEVICE_CREDENTIAL`
     * API < 30: `BIOMETRIC_WEAK or DEVICE_CREDENTIAL`
     This prevents IllegalArgumentException on API 28-29 where BIOMETRIC_STRONG|DEVICE_CREDENTIAL is unsupported.
   - `clearAuthState(context: Context)` -- removes KEY_LAST_BACKGROUND_TIME (used when disabling lock)

   Imports needed: `android.content.Context`, `android.os.Build`, `androidx.biometric.BiometricManager`
  </action>
