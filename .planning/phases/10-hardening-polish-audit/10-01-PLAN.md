**SEC-11 — Sensitive clipboard (PagesFragment.kt):**
Replace the `copyToClipboard()` method (around line 347) with a version that marks clipboard content as sensitive. After creating the ClipData, add:
```kotlin
clip.description.extras = android.os.PersistableBundle().apply {
    putBoolean(android.content.ClipDescription.EXTRA_IS_SENSITIVE, true)
}
```
No API version check needed — `EXTRA_IS_SENSITIVE` is a compile-time constant (available since compileSdk 33, this project uses 35). On API < 33 devices the extras are set but the system ignores them (correct no-op behavior). Do NOT add an `if (Build.VERSION.SDK_INT >= 33)` guard — it is unnecessary.

**SEC-12 — Accessibility data sensitive (4 XML layout files):**
Add `android:accessibilityDataSensitive="yes"` attribute to the following TextViews/ImageViews that display sensitive document content:

1. `item_document.xml` — Add to `@+id/textDocumentName` (the document name TextView, around line 40)
2. `item_recent_document.xml` — Add to `@+id/textDocName` (the document name TextView, around line 66)
3. `dialog_ocr_result.xml` — Add to `@+id/textOcrResult` (the OCR result text, around line 43)
4. `item_page.xml` — Add to `@+id/imageThumbnail` (the page thumbnail ImageView, around line 22)

The attribute was added in API 34. Since `compileSdk = 35`, it compiles without issue. On API < 34 devices the attribute is silently ignored (standard Android XML behavior). If lint warns about the attribute not being available on minSdk 24, add `tools:ignore="UnusedAttribute"` on the specific view.

**Create RootDetector.kt** in `app/src/main/java/com/pdfscanner/app/util/`:
```kotlin
object RootDetector {
    fun isDeviceCompromised(context: Context): Boolean
}
```
Implement 3 lightweight heuristic checks:
1. `checkTestKeys()` — `android.os.Build.TAGS?.contains("test-keys") == true`
2. `checkSuBinary()` — Check existence of su binary in common paths: `/system/bin/su`, `/system/xbin/su`, `/sbin/su`, `/data/local/bin/su`, `/data/local/su`, `/system/sd/xbin/su`, `/system/bin/failsafe/su`, `/vendor/bin/su`
3. `checkDebuggable(context)` — `(context.applicationInfo.flags and ApplicationInfo.FLAG_DEBUGGABLE) != 0`

Return `true` if ANY check passes. This is warn-only per SEC-14 — NEVER call `finish()` or disable functionality based on root detection.

**Add string resources** to `strings.xml`:
```xml
<string name="security_warning_title">Security Notice</string>
<string name="security_warning_rooted_message">This device appears to be rooted or running a modified system. Your scanned documents may be at higher risk of unauthorized access.\n\nFor maximum security, consider using an unmodified device.</string>
<string name="i_understand">I Understand</string>
```

**Wire root detection into MainActivity.kt:**
Add a `checkRootedDevice()` private method and call it in `onCreate()`, AFTER the biometric/lock overlay setup (after line 141 — after `cleanupStaleTempFiles()`), guarded by `!BuildConfig.DEBUG`:

```kotlin
// SEC-14: Root/debuggable detection — release builds only
if (!BuildConfig.DEBUG) {
    checkRootedDevice()
}
```

The `checkRootedDevice()` method:
1. Return early if `RootDetector.isDeviceCompromised(this)` is false
2. Read `SecurePreferences.getInstance(this)` with key `"${SecurePreferences.APP_PREFIX}root_warning_dismissed"`
3. Return early if already dismissed (value is `true`)
4. Show `MaterialAlertDialogBuilder` with title `R.string.security_warning_title`, message `R.string.security_warning_rooted_message`, positive button `R.string.i_understand` that sets the dismissed flag to `true` via `prefs.edit().putBoolean(key, true).apply()`
5. Set `setCancelable(false)` — user must acknowledge

Import `com.pdfscanner.app.util.RootDetector` and `com.pdfscanner.app.util.SecurePreferences` in MainActivity (SecurePreferences may already be imported if used elsewhere). Also import `com.google.android.material.dialog.MaterialAlertDialogBuilder`.

IMPORTANT: The `FLAG_DEBUGGABLE` check inside RootDetector will always return true in debug builds. The `!BuildConfig.DEBUG` guard in MainActivity prevents the dialog from ever appearing during development. This is the same pattern used for FLAG_SECURE in SEC-01 (locked decision).
