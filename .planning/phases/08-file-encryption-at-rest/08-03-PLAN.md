**PagesAdapter.kt** — bind() thumbnail loading (around line 233):
- Currently uses `binding.imageThumbnail.load(uri)` where `uri` is a file:// URI to a scan in app storage.
- Coil cannot read encrypted files. Replace with bitmap-first loading:
  ```kotlin
  // Decrypt scan image, then load bitmap into Coil
  val file = File(uri.path ?: return)
  if (file.exists()) {
      val bitmap = SecureFileManager.decryptToBitmap(file)
      if (bitmap != null) {
          binding.imageThumbnail.load(bitmap) {
              crossfade(true)
              placeholder(R.drawable.ic_cartoon_document)
          }
      } else {
          // Fallback: try loading URI directly (may be unencrypted pre-migration file)
          binding.imageThumbnail.load(uri) {
              crossfade(true)
              placeholder(R.drawable.ic_cartoon_document)
          }
      }
  }
  ```
- NOTE: `decryptToBitmap` already falls back to `BitmapFactory.decodeFile` if decryption fails, so the else branch covers the case where the file exists but SecureFileManager returned null (e.g., no context initialized). The bitmap loading runs on Coil's default IO dispatcher.
- IMPORTANT: `decryptToBitmap` is a blocking call. It should be called from a coroutine or on a background thread. If the adapter's bind method is on the main thread, wrap in a coroutine: use `holder.itemView.findViewTreeLifecycleOwner()?.lifecycleScope?.launch(Dispatchers.IO)` or use the existing adapter pattern. Check how the adapter currently handles async work -- if it uses Coil's built-in async (which runs on IO), we need to match that. Since Coil's `load(bitmap)` is synchronous for bitmaps but `decryptToBitmap` is IO-bound, wrap the decrypt call in a coroutine scope that the RecyclerView provides. Use `kotlinx.coroutines.CoroutineScope(Dispatchers.IO).launch` with `withContext(Dispatchers.Main)` for the Coil load.
- SIMPLER APPROACH: Use Coil's `load()` with a custom `data` parameter. Actually, the simplest working pattern is:
  ```kotlin
  lifecycleScope.launch(Dispatchers.IO) {
      val bitmap = SecureFileManager.decryptToBitmap(file)
      withContext(Dispatchers.Main) {
          binding.imageThumbnail.load(bitmap ?: uri) { ... }
      }
  }
  ```
  But RecyclerView adapters don't have lifecycleScope. Instead, pass the fragment's lifecycleScope to the adapter via constructor, or use `binding.root.findViewTreeLifecycleOwner()?.lifecycleScope`. Check what pattern the adapter already uses for async work and follow it.
