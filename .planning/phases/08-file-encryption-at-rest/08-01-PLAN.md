<task type="auto">
  <name>Task 2: Create SecureFileManager singleton with full encryption API</name>
  <files>app/src/main/java/com/pdfscanner/app/util/SecureFileManager.kt</files>
  <action>
Create `SecureFileManager.kt` as an `object` singleton in the `util` package (same pattern as SecurePreferences). The singleton manages a Tink StreamingAead keyset with Android KeyStore protection. It MUST use a SEPARATE KeyStore alias (`doc_file_master_key`) from the SecurePreferences MasterKey (`_androidx_security_master_key_`) -- this is a locked decision from STATE.md ("Biometric auth keys MUST be separate from file encryption keys").

The singleton must provide these public methods:

**Initialization:**
- `fun getInstance(context: Context): StreamingAead?` -- Double-checked locking, lazy init. Calls `StreamingAeadConfig.register()`, creates `AndroidKeysetManager` with `withSharedPref(context, "doc_file_keyset", "file_encryption_keyset")`, `withKeyTemplate(PredefinedStreamingAeadParameters.AES256_GCM_HKDF_4KB)`, `withMasterKeyUri("android-keystore://doc_file_master_key")`. Returns null on KeyStore failure (graceful degradation, same pattern as SecurePreferences fallback). If `PredefinedStreamingAeadParameters` is unavailable at compile time, fall back to `KeyTemplates.get("AES256_GCM_HKDF_4KB")`.

**Write operations (all fallback to plaintext if StreamingAead is null):**
- `fun encryptToFile(file: File, data: ByteArray)` -- Write encrypted bytes to file
- `fun encryptBitmapToFile(bitmap: Bitmap, file: File, quality: Int = 90)` -- Compress bitmap through encrypting stream
- `fun encryptPdfToFile(pdfDocument: PdfDocument, file: File)` -- Write PdfDocument through encrypting stream

**Read operations (all attempt decrypt, fall back to plaintext on failure for pre-migration files):**
- `fun decryptFromFile(file: File): InputStream` -- Return decrypting InputStream (or plain FileInputStream on failure)
- `fun decryptToTempFile(context: Context, encryptedFile: File): File` -- Decrypt to temp file in cacheDir with `dec_${UUID.randomUUID()}.tmp` name. Caller MUST delete.
- `fun decryptToBitmap(file: File): Bitmap?` -- Decrypt and decode via BitmapFactory.decodeStream, fallback to BitmapFactory.decodeFile

**In-place encryption (for CameraX post-capture and migration):**
- `fun encryptFileInPlace(file: File)` -- Read plaintext, write encrypted to `${file.name}.enc.tmp` in same parent dir, atomic rename over original. Before encrypting, attempt decrypt first -- if decryption succeeds, file is already encrypted, skip it (idempotent for crash-safe migration).

**Secure deletion (SEC-10):**
- `fun secureDelete(file: File): Boolean` -- Overwrite file content with SecureRandom bytes in 8KB chunks, call `fd.sync()`, then `file.delete()`. Fallback to plain `file.delete()` on any error. Return true if file no longer exists.

**Migration:**
- `suspend fun migrateExistingFiles(context: Context, onProgress: (current: Int, total: Int) -> Unit): Boolean` -- Check sentinel `_file_migration_complete` in SecurePreferences. If already set, return true. Collect all files in filesDir/scans/, filesDir/processed/, filesDir/pdfs/. Call `encryptFileInPlace()` for each, invoke `onProgress` callback. Write sentinel after all files processed. Return true on success.

**Testing support:**
- `internal fun resetForTesting()` -- Reset singleton (same pattern as SecurePreferences)

Use `private val ASSOCIATED_DATA = "pdfscanner_file_v1".toByteArray()` for all StreamingAead operations.

All encrypt/decrypt methods must be safe to call from `Dispatchers.IO` (no main-thread work). Do NOT encrypt temp files in cacheDir (anti-pattern from research). Log warnings on encryption failures but never crash.
