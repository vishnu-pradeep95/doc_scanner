<objective>
Move all hardcoded English text and emoji from layout XML files to strings.xml, and move all hardcoded contentDescription strings to string resources. Covers primary fragment layouts, dialog layouts, and item layouts.

Purpose: Android Lint (configured in Phase 5) treats hardcoded text as errors. Moving strings now avoids a mass-fix later. Emoji in android:text attributes is fragile — different renderers handle them inconsistently, and screen readers may misread them.
Output: Zero hardcoded android:text strings in primary layouts; all contentDescriptions reference @string/; emoji removed from text attributes.
</objective>

For each file, apply the replacements from the interfaces section above. The rule for each change:

**android:text replacements:**
Replace `android:text="[literal string with possible emoji]"` with `android:text="@string/[key]"` where [key] is the mapping from the interfaces section. The @string value must NOT contain emoji (emoji is extracted to drawable or simply removed).

**android:contentDescription replacements:**
Replace `android:contentDescription="[literal string]"` with `android:contentDescription="@string/[key]"` where [key] is the mapping from the interfaces section.

**Emoji removal rule:**
When replacing a text that had an emoji (e.g., "🎨 Style", "📷 Retake"), the string resource value has the emoji removed. The button/label communicates the action via text alone. Do NOT add the emoji back to the resource value.
