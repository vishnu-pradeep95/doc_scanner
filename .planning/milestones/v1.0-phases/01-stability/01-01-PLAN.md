<objective>
Refactor ScannerViewModel to use immutable collection patterns and SavedStateHandle for process death recovery.

Purpose: ScannerViewModel currently uses MutableList and MutableMap inside LiveData, which causes silent data loss from concurrent modification and provides zero process death recovery. This is the foundation that all fragments depend on -- it must be fixed first.

Output: A ScannerViewModel that (1) creates new List/Map instances on every mutation (BUG-04), and (2) persists page URIs, page filters, and PDF base name in SavedStateHandle (BUG-05).
</objective>

  <action>
Rewrite ScannerViewModel to accept `SavedStateHandle` as a constructor parameter and back all user-critical state with it.

**Constructor change:**
```kotlin
class ScannerViewModel(private val savedStateHandle: SavedStateHandle) : ViewModel()
```
No factory needed -- `by activityViewModels()` with `SavedStateViewModelFactory` handles this automatically when the constructor takes SavedStateHandle.

**Add companion object with keys:**
```kotlin
companion object {
    private const val KEY_PAGES = "pages"
    private const val KEY_PAGE_FILTERS = "page_filters"
    private const val KEY_PDF_BASE_NAME = "pdf_base_name"
}
```

**Pages (BUG-04 + BUG-05):**
- Replace `private val _pages = MutableLiveData<MutableList<Uri>>(mutableListOf())` with:
  `val pages: LiveData<List<Uri>> = savedStateHandle.getLiveData(KEY_PAGES, emptyList())`
- Remove the separate `_pages` backing field entirely.
- `addPage()`: `savedStateHandle[KEY_PAGES] = savedStateHandle.get<List<Uri>>(KEY_PAGES).orEmpty() + uri`
- `updatePage()`: Create new list with element replaced at index.
  ```kotlin
  val current = savedStateHandle.get<List<Uri>>(KEY_PAGES).orEmpty()
  if (index in current.indices) {
      savedStateHandle[KEY_PAGES] = current.toMutableList().also { it[index] = newUri }.toList()
  }
  ```
- `removePage()`: Create new list without element.
  ```kotlin
  val current = savedStateHandle.get<List<Uri>>(KEY_PAGES).orEmpty()
  if (index in current.indices) {
      savedStateHandle[KEY_PAGES] = current.toMutableList().also { it.removeAt(index) }.toList()
  }
  ```
- `movePage()`: Create new list with swapped elements.
  ```kotlin
  val current = savedStateHandle.get<List<Uri>>(KEY_PAGES).orEmpty().toMutableList()
  if (fromPosition in current.indices && toPosition in current.indices) {
      java.util.Collections.swap(current, fromPosition, toPosition)
      savedStateHandle[KEY_PAGES] = current.toList()
  }
  ```
- `getPageCount()`: `savedStateHandle.get<List<Uri>>(KEY_PAGES).orEmpty().size`

**Page Filters (BUG-04 + BUG-05):**
- Store as `Map<Int, String>` (enum name strings) in SavedStateHandle since FilterType is not Parcelable.
- `val pageFilters` becomes a regular LiveData derived from SavedStateHandle. Use `savedStateHandle.getLiveData<Map<Int, String>>(KEY_PAGE_FILTERS, emptyMap())` as the backing store, but expose as `LiveData<Map<Int, ImageProcessor.FilterType>>` via Transformations.map.
  ```kotlin
  private val _rawPageFilters: LiveData<Map<Int, String>> = savedStateHandle.getLiveData(KEY_PAGE_FILTERS, emptyMap())
  val pageFilters: LiveData<Map<Int, ImageProcessor.FilterType>> = _rawPageFilters.map { raw ->
      raw.mapValues { (_, name) -> ImageProcessor.FilterType.valueOf(name) }
  }
  ```
  Import `androidx.lifecycle.map` for the `map` extension.
- `setPageFilter()`:
  ```kotlin
  val current = savedStateHandle.get<Map<Int, String>>(KEY_PAGE_FILTERS).orEmpty()
  savedStateHandle[KEY_PAGE_FILTERS] = current + (pageIndex to filterType.name)
  ```
- `getPageFilter()`:
  ```kotlin
  val name = savedStateHandle.get<Map<Int, String>>(KEY_PAGE_FILTERS)?.get(pageIndex)
  return name?.let { ImageProcessor.FilterType.valueOf(it) } ?: ImageProcessor.FilterType.ORIGINAL
  ```

**PDF Base Name (BUG-05):**
- `val pdfBaseName: LiveData<String?> = savedStateHandle.getLiveData(KEY_PDF_BASE_NAME)`
- `setPdfBaseName()`: `savedStateHandle[KEY_PDF_BASE_NAME] = name?.trim()?.takeIf { it.isNotEmpty() }`

**Non-persisted state (keep as plain MutableLiveData):**
- `_currentCaptureUri`, `_pdfUri`, `_isLoading` -- these are transient session state that should NOT survive process death (camera capture is session-specific, PDF URI must be regenerated, loading state is transient).

**clearAllPages():**
```kotlin
savedStateHandle[KEY_PAGES] = emptyList<Uri>()
_pdfUri.value = null
savedStateHandle[KEY_PAGE_FILTERS] = emptyMap<Int, String>()
savedStateHandle[KEY_PDF_BASE_NAME] = null as String?
```

**CRITICAL NOTES:**
- `Uri` implements `Parcelable`, so `List<Uri>` can be stored in SavedStateHandle directly.
- `Map<Int, String>` contains only primitives, so it is Bundle-safe.
- Every mutation MUST create a new collection instance (not mutate in-place) to trigger LiveData observers and DiffUtil in ListAdapters.
- The `pages` LiveData type changes from `LiveData<MutableList<Uri>>` to `LiveData<List<Uri>>`. Any fragment code that calls `pages.value?.add()` directly will get a compile error, which is the desired outcome -- the immutable type prevents accidental mutation. However, since all fragments currently only observe and read (they use ViewModel methods to mutate), this should compile cleanly.
- Keep all existing imports. Add: `import androidx.lifecycle.SavedStateHandle`, `import androidx.lifecycle.map`.
