**PreviewFragment.kt -- BUG-06: Cap full-resolution bitmap loading:**

Read the full file. Find the method that loads the full-resolution bitmap for filter preview (research identifies it as `loadFullResBitmap` or similar). This is the method that loads the original image at full camera resolution.

The target dimensions should be 2x A4 at 144dpi = 1190 x 1684 pixels maximum. This is the same target PagesFragment.generatePdf uses via decodeSampledBitmap.

Create or modify the bitmap loading to use `BitmapFactory.Options.inSampleSize`:

```kotlin
private fun decodeCappedBitmap(context: Context, uri: Uri, maxWidth: Int = 2380, maxHeight: Int = 3368): Bitmap? {
    // First pass: decode bounds only
    val options = BitmapFactory.Options().apply { inJustDecodeBounds = true }
    context.contentResolver.openInputStream(uri)?.use { stream ->
        BitmapFactory.decodeStream(stream, null, options)
    }

    // Calculate inSampleSize
    options.inSampleSize = calculateInSampleSize(options.outWidth, options.outHeight, maxWidth, maxHeight)
    options.inJustDecodeBounds = false

    // Second pass: decode with downsampling
    return context.contentResolver.openInputStream(uri)?.use { stream ->
        BitmapFactory.decodeStream(stream, null, options)
    }
}

private fun calculateInSampleSize(rawWidth: Int, rawHeight: Int, maxWidth: Int, maxHeight: Int): Int {
    var inSampleSize = 1
    if (rawWidth > maxWidth || rawHeight > maxHeight) {
        val halfWidth = rawWidth / 2
        val halfHeight = rawHeight / 2
        while ((halfWidth / inSampleSize) >= maxWidth && (halfHeight / inSampleSize) >= maxHeight) {
            inSampleSize *= 2
        }
    }
    return inSampleSize
}
```

Replace ALL places in PreviewFragment where a full-resolution bitmap is decoded from a URI with this capped version. The max dimensions (2380x3368) are 2x the A4 PDF page size at 144dpi, providing plenty of quality for PDF output while preventing OOM on 48MP cameras (which would otherwise try to allocate 8000x6000 * 4 bytes = 192MB).

Also: ensure that after filter application, the processed bitmap is recycled when no longer needed. Check for any bitmap that is created but not recycled on the success path (not just the error path).

Add a dimension cap at the start of `applyMagicFilter()`:
```kotlin
fun applyMagicFilter(inputBitmap: Bitmap): Bitmap {
    var bitmap = inputBitmap
    // Cap dimensions to prevent OOM on high-megapixel images
    val maxDim = 3368  // 2x A4 at 144dpi
    if (bitmap.width > maxDim || bitmap.height > maxDim) {
        val scale = minOf(maxDim.toFloat() / bitmap.width, maxDim.toFloat() / bitmap.height)
        val newWidth = (bitmap.width * scale).toInt()
        val newHeight = (bitmap.height * scale).toInt()
        val scaled = Bitmap.createScaledBitmap(bitmap, newWidth, newHeight, true)
        if (scaled != bitmap) bitmap.recycle()
        bitmap = scaled
    }
    // ... rest of existing filter logic using bitmap instead of inputBitmap
}
```
